ATM - Dashboard Overview

  1. Dashboard

คู่มือการใช้งานระบบ Check-In, Check-Out เอกสาร

คู่มือการใช้งาน

โดยทั่วไปแล้วระบบ Check-In / Check-Out จะมี State หลักๆอยู่ 3 States ได้แก่

- UnAssigned คือ ไฟล์ที่อยู่เฉพาะบนเครื่อง Local ฝั่ง Server จะยังไม่รู้จัก

- Added คือ ไฟล์ที่อยู่เฉพาะบนเครื่อง Local แต่ตอนนี้ Server รู้จักแล้ว

- Committed คือไฟล์ที่อยู่บนเครื่องเรา และส่งไปเก็บยัง Server แล้ว

สมมติว่า เราต้องการนำไฟล์ test.txt เข้าไปเก็บยัง Server ต้องทำยังไง

1. Copy ไฟล์ ไปวางใน Folder ที่เราได้ทำการ Checkout มาเมื่อข้างต้นซึ่งก็คือ Base_Line_Documents

2. คลิ๊กขวาที่ไฟล์ ตามตัวอย่างคือ test.txt แล้ว Add ตามภาพเพื่อให้ Server รู้จักไฟล์นี้

 

3. ทำการส่งไฟล์ขึ้น Server โดยการ คลิ๊กขวาที่ไฟล์ แล้วเลือก Commit

4. ใส่ Comment นิดหน่อย เพื่อจะได้รู้ว่าไฟล์นี้มีการแก้ไขอะไร แล้วกด OK

เพียงเท่านี้ไฟล์ของเราก็จะถูกส่งไปยัง Server พร้อมแชร์ให้คนอื่นแล้วครับ

Note: ทดลองทำการลบไฟล์ Test.txt ที่เครื่องทิ้งดู แล้วทดลองกด Update ตามรูป

เราก็จะได้ไฟล์กลับคืนมา....

ข้อดีของระบบนี้ก็คือ

1. ถ้าไฟล์ไหนถูก Commit ไปยัง Server แล้ว จะสามารถ Track กลับคืนมาได้ทุก Version ไม่สูญหายแน่นอน

2. กรณีที่มีการ Checkout ไฟล์เดียวกันไปใช้งานพร้อมๆกัน ถ้าคนแรก Commit ไฟล์กลับไปก่อน คนที่สองจะไม่สามารถ Commit ไฟล์ได้

ช่วยเรื่องการแก้ไขเอกสารทับกันได้

3. สามารถ Track กลับได้ตลอดเวลาว่าเอกสารเล่มไหน ใครเป็นคน commit เมื่อไหร่ยังไง

ข้อเสีย

1. ไฟล์จะถูกเก็บอยู่ที่ Local PC ของเราด้วยทำให้เปลือง Disk

วิธีแก้คือ ให้ลบโฟล์เดอร์ย่อยๆที่ไม่ได้ใช้ทิ้งไปเลย จะใช้เมื่อไหร่ค่อย Checkout มา

เช่น ใน14Base จะมีทั้ง RA, Design, Test, Deploy ถ้าเราใช้แค่ Deploy โฟล์เดอร์อื่นก็ลบไป
